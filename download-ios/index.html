Which option should I use?
  • IPA (Sideload) = you install it using a signer app (ESign / KSign / other signer).
  • Direct Install = install from Safari, then you may need to trust the app.
  • If Direct Install 1 doesn’t work, try Direct Install 2.
  • If you updated, check Updates after installing.
How to install using the IPA (Sideload)

The IPA method requires a signer app such as ESign, KSign, or any other iOS signer. Signer setup is not our department — but here’s the basic workflow so you know what to do:

Step 1

Tap Download iOS IPA and save the file.

Step 2

Open your signer app (ESign / KSign / other) and import the IPA.

Step 3

Use the signer app to install. If something fails, check Support.

Why use the IPA method?

IPA sideloading is useful when direct install is unavailable. It also gives you a consistent install flow if you already use a signer app.

How to install using Direct Install

Direct install is the simplest: tap Direct Install 1 or 2 and follow the install prompts. After installing, iOS may say the developer is untrusted — that’s normal for direct installs.

Step 1

Tap Direct Install 1. If it doesn’t work, use Direct Install 2.

Step 2

Accept the install popup. Wait for the app icon to appear.

Step 3

If it says “Untrusted Developer”, trust the app (steps below).

How to trust the app after Direct Install

Go to Settings → General → VPN & Device Management, then find the profile / enterprise app and tap Trust. After trusting, reopen the app.

Direct Install button is grey or does nothing

Try the other direct link, refresh Safari, switch Wi-Fi/mobile data, then retry. If it still doesn’t work, use the IPA method.
